Node.js SQL etc.

Node.js - Used for taking your webdevelopment techniques from the browser and takes them to the application side/back end/ allows you to write code using the Command Line and allows for the usage of packages which can make servers and aid overall development - KING OF BACKEND** - to run a js file from cmd line just write - 'node index.js' from the appropriate part of the termina;

NPM (Node Package Manager) - Allows for the downloading and usage of a vast array of packages using Node.js, i.e. using CMD for example, from the command line, navigate to the appropriate part of your system and type npm install nodemon or npm install -express etc. You can also use the website to install any packages that you may need. These packages then need to the 'Required' in the .JS file i.e. const express = require('express'); followed by const app = express(); for express.

Express.js - Using NPM, installing this package to use with node is essential for moving code to a server side package - use node to run the js file and from here if the file is setup using express the code will be moved to a server using the listen method with post, get, use, send and more method, api docs can be found online.

Nodemon - used in combination with Express will automatically refresh server with updates as they are saved - once installed with npm , simply run the express server code with 'nodemon index.js' and go to the server location and the code with refresh with each save mode to the js code, saving time to start and stop all the time

Body-parser - Essential to use this in combination with Express.js - set up using 'const bodyParser = require('body-parser');' & 'app.use(bodyParser.urlencoded({extended: true}));' - from here it is used from within the POST method for express and with the Request passed you can access the outputs from the body, i.e. req.body will return all outputs from the HTML or req.body.h1 will return the element output
